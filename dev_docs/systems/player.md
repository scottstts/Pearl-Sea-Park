# Player, physics & interaction (S5)

- **Physics**: Rapier world in `physics/physicsWorld.ts`; plain gravity (the sea is air). Terrain collider is a 128² heightfield sampled from `terrainHeight` — **column-major** (`heights[col*(nrows+1)+row]`), verified by a downward-raycast self-check under `?debug` (tolerance 1.5 m grid interpolation). `addStaticBox()`/`addStaticCylinder()` are the helpers structures use; **anything solid a guest can reach must add its own collider** — the capsule already blocks every collider, so "walking through mesh" is always a missing collider, not a controller bug. Ride superstructures built inline are the usual gap (the Great Wheel raking legs and the Torrent track piers now add foot/pillar cylinders); flora, coral, wildlife, and tabletop props stay walk-through by design.
- **Player** (`player/player.ts`): kinematic capsule + Rapier character controller (autostep 0.45/0.25, snap-to-ground, slope 52°), walk **2.4** / brisk **4.65** m/s (1.5× the original stroll/brisk), eased acceleration, near-imperceptible bob. Look = pointer lock; movement keys work without lock (automation relies on this). **yaw 0 faces −z = north = toward the park**; camera and movement share yaw so they can never disagree.
- **Underwater jump** (Space): a diver's push-off, not a platformer hop. Effective gravity **follows the medium** — `SWIM_GRAVITY` 2.6 below the waterline, `AIR_GRAVITY` 9.81 above (`JUMP_SPEED` 3.15 ⇒ ~1.9 m apex, ~2.4 s hang). Gated on `submerged && grounded && controlEnabled`, which is exactly "in the underwater park, outside the elevator, not in a ride car" (rides null control). `submerged` is driven by the `sea/waterline-crossed` event, not a y-test. **Snap-to-ground is toggled off for the arc** (`disableSnapToGround()` on launch, re-enabled on landing) or Rapier glues the leap back to the floor; `placeAt()` also clears the jump/queue and re-arms snap. Jump is edge-triggered (`event.repeat` filter) and buffered jumps are dropped whenever control is borrowed.
- Rides/seats borrow control via `player.controlEnabled = false` + `placeAt()` on return. **`inputFrozen`** is a *separate* modal freeze (teleport menu/transition): it stops walking + look without touching `controlEnabled`, so a pause captured during the freeze cannot strand control on a borrowed value.
- **Pause card** (`ui/pauseCard.ts`): Escape's pointer-lock release opens the
  back of Golden Ticket No. 1, freezes fixed simulation and authored time,
  suspends Web Audio so scheduled music cannot drift, preserves ride-owned player control, and offers resume, persistent master
  volume, or Auto/Gentle/Fine/Grand quality. Tier changes reload intentionally
  because wildlife, water, shadow, and VFX storage budgets are allocated once
  before world initialization.
- **Interaction** (`player/interact.ts`): registry of `{position, radius, prompt, key, onInteract, enabled}`; nearest eligible within radius + view cone shows the single serif caption (`.prompt` CSS). All gates/games/benches register here — never build bespoke prompt UI. `interaction.suspended` mutes prompts **and** the key handler while a modal (the teleport menu) is open — the loop keeps running (teleport does not pause), so without it an E press meant for the menu would fire a nearby bench/gate.
- **Teleport network** (`player/teleport.ts`): every facility marker (incl. the added `park-entrance` node beside the Descent Bell landing) is a node. Approaching shows "Teleport"; E opens a serif menu of all sites (↑/↓ choose, Enter travel, Esc back), and the move plays under a full-screen deep-blue fade (`.teleport-fade`, the visit's opening field) with a centered "Teleporting" notice. Landing point is `SPAWN_DIST` (3.6 m) in front of the sign along its approach ray, turned to face the board; ground is `terrainHeight` (park markers sit near-flush on the seabed). The fade is a 5-phase state machine (`idle→menu→out→hold→in`); `placeAt()` happens at full opacity so the camera cut is unseen. Esc cancels *and* raises the pause card (Esc always releases the pointer) — accepted, since Esc is the game's universal "out".
- **Seats** (`player/seats.ts`): smooth eased camera in/out (0.9 s), any move key leaves, `player.placeAt(exit)` on stand-up. Rides reuse `enter/leave`. (Space leaves a seat *and* would queue a jump; the borrowed-control path drops the queued jump so standing up never launches.)
- **Held items** (`player/heldItems.ts`): **state only — nothing renders**. Scott's ruling (2026-07-10 quality pass): the game is a clean first-person POV with no player body and no camera-attached props; the old hand rig's ticket card read as a floating artifact. The system still tracks ticket stamps, pressed pennies, prizes, and the current item, and still emits `ticket/completed`, so rides/games keep their award/query API. If a future feature needs visible possessions, propose it to Scott first — do not re-parent meshes to the camera.
- `?debug` exposes `window.__pearl = { ctx, registry, qualitySelection, postcardAudit }` for console/automation inspection. Canvas datasets also carry performance, exposure, shadow, fountain, wildlife, games, and postcard snapshots.
- Mode rule in main.ts: `?view=` → DevOrbit inspection camera; no view → player.
